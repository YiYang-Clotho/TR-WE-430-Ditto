What found easy? difficult? how to fix?

In this project, user stories design is easier than other parts. The most difficult part of our project is function implementing. Though spring boot is one of the best Java frameworks in the world. We still met some problems in how to set up the authority management. For example, we try to use jwt (java web token) to authorize the users. The most common solution is to put the token to the header using Axios (an HTTP request lib) and send it to the server, but we use thymeleaf to build our interface rather than react.js or vue.js. Therefore it is hard to put the web token to the request’s header and send it to the spring boot. After the discussion in the meeting, we chose to put the token to request cookies and set an interceptor function to get requests and verify the token in the cookie. Another unexpected and difficult part is to implement file upload functionality. Firstly, we prefer to store recipe pictures and avatars locally. However, we can not share recipes. So we selected a s3 bucket and learned AWS SDK to store pictures online.

expecting Challenges

One of the challenges we expect is feature implementation including data delivery between the browser and the server. Thymeleaf and HTML only support “get” and “post” HTTP requests. So we chose Axios to send HTTP requests from the front end. We also did lots of research to allow the server to receive data properly. Another challenge we expect is interaction functionality. We find lots of material and tutorials to learn javascript because javascript is very useful to build a dynamic web page and improve user experiences such as error prevention and animation. The last challenge we expect is the user interface. We must know HTML and CSS a lot to build a user-friendly interface, so we learn flex to plan web pages’ layouts and learn bootstrap to build a proper interface for our system.

what we learned

We learned a lot from this course. First is communication. In order to implement our system “Cookiez”, we have to collaborate to design the system, to discuss user stories, to distribute tasks and so on. We have a weekly meeting which allows us to share what we have done and review the work we have done. This is an excellent chance to exercise our ability to describe something. Besides, we learned how to take advantage of AWS to help us to develop our project. We realize what ACI(Access Control Information) is and can set access control for the public. In terms of EC2, we do lots of research to deploy our project on a Linux server. Finally, we chose docker as our solution because docker is popular in the industry and there are many tutorials about docker. We also learn RDS in aws because we need to synchronize our data.

hindsight

If there is one thing over again, we hope we can design user stories carefully and precisely with more acceptance criteria. We should learn more web programming technologies and concepts like spring boot, MVC, restful and so on. JavaScript is also very important but we did not use this web language skillfully.
