Cookiez

1.Description

In the current Covid-19 epidemic situation in most countries, especially in Australia, people are restricted from going out and that greatly affects the quality of their meals. The project we decided to implement is a food and recipe application that organizes a list of recipes based on the ingredients in the user's kitchen. It is expected to solve the problem of not knowing how to use the ingredients at home and providing ideas on what to cook.

High level user story:
As a user, I want to know which recipe I can use to cook with ingredients in my pantry and I can order online if I don’t have some of the required ingredients
so that I can better prepare daily meals for me and my family and save the time as well.

2.Demonstrable Outcomes

2.1 Project Motivation

This is an upgrade project from the existing applications. We have found their missing points and decided to improve them with this recipe application project. They are not only suitable for all ages but also provide the best support for them. Other apps like Tasty or Yummly only offer available menus and force users to follow them in order to cook the best food. Our app is pretty much the same, which makes a difference and is also the purpose of the app is to help users find the recipe they want to cook with everything in their pantry.

This is a project that oriented towards database management. We learned about them in year 1 so we can properly apply it to this project. Plus, this project is about web editing and designing as the main part of this course, therefore, each person needs to develop their own coding and managing skills for each stage of the app development.

3.3 Risks

    team dynamics risk

  1. Risk: Do not have enough time to finish all the features.
    Example: Our group may face hard technological problems.
    Mitigation:Ask the lecturer or the tutor. More communication to decrease the number of unreachable features.

  2. Risk: Suddenly or unpredictable requirements are adding
    Example: we may forget to add a vital function to the application.
    Mitigation:do more communication and learn more about how to : Build a system

  3. Risk: Limited knowledge reserve lead to the delay our project’s progress.
    Example: we may have no idea to achieve the background side.
    Mitigation:learn more or delete difficult features

    4. project risks

  4. Risk: Recipe content
    Example: recipe content cannot be shown properly.
    Mitigation:learn more html and css technology.

  5. Risk: Concurrency control
    Example: Overloaded server
    Mitigation:Minimize main database but still keep the flexibility and the algorithms that used for the main function.

  6. Risk: Content control/authorization
    Example: The workload for administrators is huge.
    Mitigation:Implement the authorization program which can check each post that user going to upload, and the checking process will be taken over by cooperated chefs
