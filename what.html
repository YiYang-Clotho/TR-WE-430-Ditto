Cookiez

1.Description

In the current Covid-19 epidemic situation in most countries, especially in Australia, people are restricted from going out and that greatly affects the quality of their meals. The project we decided to implement is a food and recipe application that organizes a list of recipes based on the ingredients in the user’s kitchen. It hopes to solve the problem of not knowing how to use the ingredients at home and providing ideas on what to cook it when you have no specific ideas. By doing so, we hope to save million tons of food wastage issue, support user to save money from ordering food from restaurants and provide a user-friendly interface that users of all ages can enjoy.

High level user story:
As a user, I want to know which recipe I can use to cook with ingredients in my pantry and I can order online if I don’t have some of the required ingredients
so that I can better prepare daily meals for me and my family and save the time as well.

2.Demonstrable Outcomes

2.1 Project Motivation

This is an upgrade project from the existing applications. We have found their missing points and decided to improve them with this recipe application project. They are not only suitable for all ages but also provide the best support for them. Other apps like Tasty or Yummly only offer available menus and force users to follow them in order to cook the best food. Our app is pretty much the same, which makes a difference and is also the purpose of the app is to help users find the recipe they want to cook with everything in their pantry.

This is a project that oriented towards database management. We learned about them in year 1 so we can properly apply it to this project. Plus, this project is about web editing and designing as the main part of this course, therefore, each person needs to develop their own coding and managing skills for each stage of the app development.

2.2 Minimum Viable Features (MVF)

  • Personal account managing system:
    Users should be able to create new accounts, log in, and log out.
  • Slider:
    The main page will have a picture slider that shows recommending recipes.
  • Data collection and recommendation
    On the website, there should be a table to collect the user’s kitchen remains so that suitable dishes can be recommended accordingly.
  • Recipe information:
    Below the slider, there will be a list of recipes with pictures. Users can click on the picture for further information.
  • Step-by-step instructions:
    Each recipe will have detailed descriptions of ingredients and cooking methods. Besides, step-by-step instructions with pictures will be applied to help the user to cook.
  • Tags and searching:
    To help users find their suitable recipes, tags will be used in recipes. Those tags may contain key ingredients. Users will have a more pleasing searching experience because tags help them to target recipes more precisely.
  • Upload and manage posts:
    Users are expected to upload their own posts and be able to modify or delete them.

2.3 Extended Viable Features (EVF)

  • Recipe rating and comment:
    Users are free to rate and comment on any recipe.
  • Health risks and precaution (Allergy Prevention):
    Recipe posts could have ingredient warnings to inform people with certain allergies of potential risks.
  • Collection and Favourite function:
    The collection and favourite function can help users collect their favorite recipes
  • Substitution:
    The website could have a function to detect missing ingredients and offer substitution suggestions.
  • Shopping helper:
    When missing ingredients are detected, the website could search nearby shops for supplement or even help users order them online.
